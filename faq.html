FAQ

Common setup and usage questions

Quick answers for the most common questions around setup commands, ticket access, vouches, and deployment.

Questions

Do I need a database server for the docs site?

No. The docs site is fully static and only uses HTML, CSS, and JavaScript.

Does the bot still need its existing runtime config?

Yes. The docs site is separate from the Discord bot runtime and does not replace the bot’s environment variables.

How do I restrict a ticket type to certain members?

Open /setuptickets, select the ticket type, switch the role target to Allowed Opener Roles, and save the roles there.

How do I switch vouch posts between embed and plain text?

Use /setupvouch and click the embed toggle button.

Can GitHub Pages host this site without any build step?

Yes. Commit the files directly and point GitHub Pages at the repository root or the branch that contains them.

What happens if a configured channel or role gets deleted?

The setup view surfaces missing references and the live systems fail gracefully instead of crashing the bot.

Troubleshooting

Note: If navigation styles or code copy buttons do not appear locally, make sure styles.css and script.js are in the same folder as the HTML files. 
Quick checks
- Confirm relative links still point to local files
- Open the browser developer tools for missing asset errors
- Verify GitHub Pages is publishing the correct branch and folder
- Re-run slash command registration after major command changes
