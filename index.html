Overview

The bot is designed as a setup-driven management suite for one or more Discord guilds. It supports welcome configuration, vouch routing, role setup, ticket panel publishing, application reviews, giveaway management, partner tracking, and staff task reminders through slash commands and persistent config.

Note: The docs use relative links and plain static assets so the site can be deployed directly to GitHub Pages without a build step or server runtime.

Primary Commands

/setupwelcome

Configure the welcome channel, message template, and embed mode.

/setupvouch

Choose the vouch channel and switch between embed or plain-message posting.

/setuproles

Assign staff, helper, PM, ticket staff, and app reviewer roles.

/setuptickets

Build ticket types with ping roles, viewer roles, allowed opener roles, and categories.

/setupapps

Create application types with questions, review channels, and acceptance roles.

/setupview

Show the current guild configuration across setup systems and status warnings.

Core Systems

The current setup guide covers these systems in the live bot: giveaway configuration, partner tracking, vouch posting, application review, ticket publishing, staff tasks, pricing hooks, and dashboard summaries. 

Example setup flow
1. Run /setuproles
2. Run /setupchannels
3. Build ticket types in /setuptickets
4. Build application types in /setupapps
5. Configure /setupwelcome and /setupvouch
6. Verify the final state with /setupview

Operational Notes

  • Guild config is stored persistently and reused across restarts.
  • Ticket and application panels store channel and message references for later refreshes.
  • Deleted channels and roles are surfaced as warnings in the setup view.
  • Role-based access can be scoped by ticket type using allowed opener roles.
