Sample API

Example endpoints for exports and integrations

The bot itself is command-driven, but these example endpoints show how a companion API could expose setup data, ticket configuration, and vouch submissions for dashboards or external tools.

Overview

These endpoints are documentation examples. They are included so the site looks complete and can serve as a design reference if you later expose setup data through a real HTTP service.

GET /guild-config/:guildId

Return the persisted configuration for a single guild.

GET /guild-config/123456789012345678
Accept: application/json
{
  "guildId": "123456789012345678",
  "welcome": {
    "enabled": true,
    "channelId": "111111111111111111",
    "embedMode": true
  },
  "vouch": {
    "enabled": true,
    "channelId": "222222222222222222",
    "embedMode": false
  }
}

POST /guild-config/:guildId/ticket-types

Create or update a ticket type definition for a guild.

POST /guild-config/123456789012345678/ticket-types
Content-Type: application/json
{
  "name": "Billing Support",
  "style": "danger",
  "categoryId": "333333333333333333",
  "pingRoleIds": ["444444444444444444"],
  "viewerRoleIds": ["555555555555555555"],
  "accessRoleIds": ["666666666666666666"],
  "openMessage": "A billing specialist will join shortly."
}

POST /guild-config/:guildId/vouches

Submit a vouch entry that mirrors the bot’s current /vouch flow.

POST /guild-config/123456789012345678/vouches
Content-Type: application/json
{
  "authorId": "777777777777777777",
  "targetUserId": "888888888888888888",
  "message": "Quick responses and clean setup."
}
Warning: If you ever implement a real endpoint, keep the Discord-side permission checks and guild scoping consistent with the slash command behavior.
